# SkufKeeper

Приложение для учёта вещей и мест хранения: шкафы, ящики с инструментами, полки и т.д.

## Возможности

- **Места хранения** — создавайте места с названием, описанием и фото; внутри места можно добавлять вложенные места (полки, ящики)
- **Объекты** — добавляйте вещи в места с фото, описанием и количеством
- **Поиск** — быстрый поиск объектов по названию
- **Статистика** — сводка по количеству мест и объектов, дерево структуры хранения
- **Адаптивный интерфейс** — удобно на ПК и iPhone
- **PWA** — можно добавить на домашний экран iPhone как приложение

## Запуск

```bash
npm install
npm run db:push
npm run dev
```

Откройте [http://localhost:3000](http://localhost:3000)

## Доступ с iPhone

1. Запустите сервер на компьютере в локальной сети:
   ```bash
   npm run dev -- -H 0.0.0.0
   ```
2. Узнайте IP компьютера (например `192.168.1.100`)
3. Добавьте в `.env` (скопируйте из `.env.example`):
   ```
   NEXT_PUBLIC_APP_URL="http://192.168.1.100:3000"
   ```
   **Важно:** укажите IP компьютера, не `localhost` — иначе QR не откроется с телефона.
4. На iPhone откройте Safari: `http://192.168.1.100:3000`
5. Нажмите «Поделиться» → «На экран Домой» для установки как приложения

## QR-коды

- При **создании места** автоматически генерируется уникальный QR-код (проверяется на дубликаты в базе)
- QR-код также отображается на странице места хранения
- Распечатайте или скачайте PNG и наклейте на шкаф/ящик
- В приложении откройте **Сканер** и отсканируйте код — откроется содержимое места
- На iPhone по HTTP используйте **«Сфотографировать QR-код»** — живая камера работает только по HTTPS

## Деплой на Railway (HTTPS)

1. Подключите репозиторий [github.com/micvant/SkufKeeper](https://github.com/micvant/SkufKeeper)
2. **Volume** → mount path: `/data`
3. **Variables**:
   ```
   DATABASE_URL=file:/data/dev.db
   NEXT_PUBLIC_APP_URL=https://skufkeeper-production.up.railway.app
   ```
   (URL возьмите из **Settings → Networking → Public URL**)
4. **Удалите** кастомный Start Command в UI — используется `railway.toml`
5. **Redeploy**

### Если изменения с GitHub не появляются на сайте

1. **Проверьте деплой:** Railway → сервис → **Deployments** — последний должен быть зелёным и свежим (после вашего push).
2. **Ручной перезапуск:** Deployments → **⋯** у последнего деплоя → **Redeploy**.
3. **Источник кода:** Settings → **Source** — репозиторий `micvant/SkufKeeper`, ветка `main`, авто-деплой включён.
4. **GitHub Actions (рекомендуется):** добавьте секреты `RAILWAY_TOKEN` и `RAILWAY_SERVICE_ID` — см. `.github/workflows/deploy-railway.yml`. После этого каждый push в `main` будет деплоить принудительно.
5. В логах деплоя при старте должна быть строка `Build: 3f51525` (короткий hash коммита).

## Стек

- Next.js 15 + React 19
- TypeScript
- Tailwind CSS 4
- SQLite + Prisma
- Локальное хранение фото в `public/uploads/`

## Структура данных

```
Место хранения (StorageLocation)
  └── Объекты (Item)
        ├── название
        ├── описание
        ├── количество
        └── фото
```
